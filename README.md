## üöÄ Quick Start

### Prerequisites

- **Ruby 3.x** - Jekyll runtime
- **Bundler** - `gem install bundler`
- **Python 3.11+** - Content transformation
- **Node.js** - Contentful CLI (optional, for schema import)

### Backend Setup (Contentful ‚Üí Jekyll)

#### 1. Install Python Dependencies

```bash
# Create virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r scripts/requirements.txt
```

#### 2. Configure Environment Variables

Create a `.env` file in the project root:

```bash
# Contentful Configuration
CONTENTFUL_SPACE_ID=co4wdyhrijid
CONTENTFUL_ACCESS_TOKEN=your_delivery_api_token_here
CONTENTFUL_PREVIEW_TOKEN=your_preview_api_token_here
CONTENTFUL_MODE=production
```

**‚ö†Ô∏è SECURITY:** Never commit `.env` to git! It's already in `.gitignore`.

**Getting Contentful Tokens:**
1. Log into [Contentful](https://app.contentful.com)
2. Navigate to Settings > API keys
3. Create or copy **Content Delivery API** token (production)
4. Create or copy **Content Preview API** token (drafts)

#### 3. Import Contentful Content Model

```bash
# Install Contentful CLI globally
npm install -g contentful-cli

# Authenticate
contentful login

# Import content types (15 schemas)
./push-contentful-schemas.sh
```

#### 4. Transform Contentful Content

```bash
# Fetch and transform content
python scripts/contentful_to_jekyll.py

# This will:
# - Fetch blog posts, profile, header, footer from Contentful
# - Transform to Jekyll markdown (_posts/) and YAML (_data/)
# - Handle both EN and ES locales
# - Complete in ~5-10 seconds locally
```

### Local Development (Rapid Iteration)

**For instant content preview:**

```bash
# Terminal 1: Start Jekyll with live reload
bundle exec jekyll serve --livereload --incremental

# Terminal 2: Re-run transformation after Contentful edits
python scripts/contentful_to_jekyll.py
```

**Workflow:**
1. Edit content in Contentful (use Preview mode for drafts)
2. Run Python script (~5-10 sec)
3. Jekyll auto-rebuilds (~3-15 sec)
4. Browser refreshes automatically
5. **Total cycle: ~10-25 seconds** (80% faster than CI/CD)

**Preview Mode (Draft Content):**

```bash
# Edit .env
CONTENTFUL_MODE=preview

# Run transformation
python scripts/contentful_to_jekyll.py

# Draft posts will be fetched and transformed
```

### Frontend-Only Development

If you just want to work on Jekyll templates/styles:

```bash
# Install Jekyll dependencies
bundle install

# Run Jekyll locally
bundle exec jekyll serve

# View site at http://localhost:4000
```

### Build for Production

```bash
# Transform content
python scripts/contentful_to_jekyll.py

# Build Jekyll site
bundle exec jekyll build
```

Generated site will be in `_site/` directory.

## üìù Content Management

Content is managed through **Contentful CMS** and transformed to Jekyll-compatible files via Python scripts.

**Content Types:**
- Blog Post (blogTemplate)
- Profile (profile)
- Header Navigation (orHeader)
- Footer (orFooter)
- SEO Metadata (seo)
- Components (cards, carousels, images, quotes, etc.)

**Transformation Flow:**
1. Content creator publishes in Contentful
2. Webhook triggers GitHub Actions
3. Python scripts fetch and transform content
4. Jekyll builds static site
5. Deploy to GitHub Pages
6. **Target:** < 5 minutes from publish to live

## üîÑ CI/CD Automation

### GitHub Actions Workflows

**Production Deploy** (`.github/workflows/production-deploy.yml`)
- **Triggers:** Push to main, manual dispatch, Contentful webhook
- **Mode:** Delivery API (published content only)
- **Steps:** Python transformation ‚Üí Jekyll build ‚Üí GitHub Pages deploy
- **Caching:** pip dependencies + Jekyll gems
- **Target:** < 5 minutes total

**Preview Deploy** (`.github/workflows/preview-deploy.yml`)
- **Triggers:** Manual dispatch only
- **Mode:** Preview API (draft content)
- **Use Case:** Review unpublished content before going live
- **Note:** Temporarily overwrites main site (simplified MVP approach)

### GitHub Secrets Configuration

Add these secrets in: Repository Settings > Secrets and variables > Actions

```
CONTENTFUL_SPACE_ID=co4wdyhrijid
CONTENTFUL_ACCESS_TOKEN=<your-delivery-api-token>
CONTENTFUL_PREVIEW_TOKEN=<your-preview-api-token>
```

### Contentful Webhook Setup

To auto-deploy on content publish:

1. **Contentful Dashboard** ‚Üí Settings > Webhooks
2. **Create Webhook:**
   - **Name:** GitHub Actions Production Deploy
   - **URL:** `https://api.github.com/repos/{owner}/{repo}/dispatches`
   - **Method:** POST
   - **Headers:**
     - `Authorization: Bearer YOUR_GITHUB_PAT`
     - `Accept: application/vnd.github+json`
   - **Content Type:** application/json
   - **Payload:**
     ```json
     {
       "event_type": "contentful-publish"
     }
     ```
   - **Triggers:** Entry.publish for content types: blogTemplate, profile, orHeader, orFooter
   - **Retry:** Enable automatic retries (Contentful retries 3 times on failure)

3. **GitHub Personal Access Token (PAT):**
   - Generate at: GitHub Settings > Developer settings > Personal access tokens > Tokens (classic)
   - Scope required: `repo` (Full control of private repositories)
   - Store securely in password manager
   - Use in Contentful webhook Authorization header

**Webhook Activity:**
- View delivery status: Contentful > Settings > Webhooks > Activity log
- Debug failures: Check GitHub Actions workflow runs
- Manual trigger: Actions > Production Deploy > Run workflow

### Build Time Monitoring

Python script monitors and logs build performance:
- **Target:** < 2 minutes for Python transformation
- **Warning:** Logs if > 2 minutes (still completes)
- **Critical:** Logs if > 4 minutes (approaching 5-min limit)
